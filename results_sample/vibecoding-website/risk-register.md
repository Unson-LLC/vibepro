## リスク詳細

### R-001: 外部リソース（プレースホルダー画像）への依存

| 項目 | 内容 |
|------|------|
| カテゴリ | 可用性 |
| 深刻度 | Medium |
| 対応優先度 | 1（商用化前に対応推奨） |
| ステータス | 未対応 |

#### 概要
プロフィール画像に外部サービス（placehold.jp）のプレースホルダー画像を使用しています。外部サービスの障害時やサービス終了時にサイトの見た目に影響が出る可能性があります。

#### ビジネスへの影響
- 影響範囲: サイト訪問者全体
- 想定被害: プロフィール画像が表示されない、サイトの信頼性低下

#### 技術詳細
- 検出箇所: `index.html:224`
- 検出内容: `<img src="https://placehold.jp/150x150.png" ...>`

#### 推奨対策
1. 実際のプロフィール画像ファイルを用意する
2. 画像ファイルをローカルに配置し、相対パスで参照する
3. 適切なサイズ・フォーマット（WebP推奨）で最適化する

#### 対応工数目安
- 工数: 0.5時間
- 難易度: 低

### R-002: ダミーURLおよび連絡先の残存

| 項目 | 内容 |
|------|------|
| カテゴリ | コンテンツ |
| 深刻度 | Low |
| 対応優先度 | 2（商用化前に対応必須） |
| ステータス | 未対応 |

#### 概要
問い合わせフォームURL、メールアドレス、SNSリンクがダミー値のままになっています。本番公開時にはユーザーが正しい連絡先に到達できない状態です。

#### ビジネスへの影響
- 影響範囲: 問い合わせを試みる訪問者全体
- 想定被害: 機会損失（問い合わせの取りこぼし）、信頼性の低下

#### 技術詳細
- 検出箇所: `index.html`
  - 298行目: `https://forms.gle/dummy`
  - 311行目: `mailto:dummy@example.com`
  - 323行目: `https://github.com/dummy`
  - 330行目: `https://x.com/dummy`
  - 337行目: `https://linkedin.com/in/dummy`

#### 推奨対策
1. Google Formsで問い合わせフォームを作成し、URLを更新
2. 実際のメールアドレスに更新
3. 各SNSプロフィールの実URLに更新

#### 対応工数目安
- 工数: 0.5時間
- 難易度: 低

### R-003: 配信不要ファイルの混在

| 項目 | 内容 |
|------|------|
| カテゴリ | 運用 |
| 深刻度 | Low |
| 対応優先度 | 3（デプロイ時に対応） |
| ステータス | 未対応 |

#### 概要
開発用ファイル（node_modules、package.json等）がプロジェクト内に存在します。これらを誤って本番環境にデプロイすると、不要なファイルが公開される可能性があります。

#### ビジネスへの影響
- 影響範囲: 限定的
- 想定被害: 不要なファイルの公開（セキュリティリスクは低）、ストレージの無駄使い

#### 技術詳細
- 検出箇所:
  - `node_modules/`（開発用パッケージディレクトリ）
  - `package.json`（パッケージ定義）
  - `pnpm-lock.yaml`（ロックファイル）
  - `.git/`（Gitディレクトリ）
  - `.gitignore`（Git設定）

#### 推奨対策
1. デプロイ時に必要なファイルのみを含むディレクトリを作成
2. deploy-plan.md の手順に従い、必要なファイルのみをデプロイ
3. CI/CDパイプラインで自動的に不要ファイルを除外する設定を検討

#### 対応工数目安
- 工数: 0.25時間
- 難易度: 低
