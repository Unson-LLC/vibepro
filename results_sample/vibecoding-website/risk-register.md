## リスク詳細

### R-001: ダミーデータの残存

| 項目 | 内容 |
|------|------|
| カテゴリ | コンテンツ |
| 深刻度 | Medium |
| 対応優先度 | 1（商用化前に対応推奨） |
| ステータス | 未対応 |

#### 概要
お問い合わせフォーム、SNSリンク、メールアドレスにダミーデータ（dummy）が使用されています。本番公開前に実際の情報に更新する必要があります。

#### ビジネスへの影響
- 影響範囲: ユーザー全体
- 想定被害: 問い合わせができない、信頼性の低下

#### 技術詳細
- 検出箇所:
  - index.html:298 `https://forms.gle/dummy`
  - index.html:311 `dummy@example.com`
  - index.html:323 `https://github.com/dummy`
  - index.html:330 `https://x.com/dummy`
  - index.html:337 `https://linkedin.com/in/dummy`

#### 推奨対策
1. 実際のGoogle FormのURLに更新
2. 実際のメールアドレスに更新
3. 実際のSNSプロフィールURLに更新

#### 対応工数目安
- 工数: 0.5時間
- 難易度: 低

### R-002: 外部画像サービスへの依存

| 項目 | 内容 |
|------|------|
| カテゴリ | 可用性 |
| 深刻度 | Low |
| 対応優先度 | 2（商用化後でも可） |
| ステータス | 未対応 |

#### 概要
プロフィール画像にプレースホルダーサービス（placehold.jp）を使用しています。外部サービスが停止した場合、画像が表示されなくなります。

#### ビジネスへの影響
- 影響範囲: 自己紹介セクションのみ
- 想定被害: プロフィール画像が表示されない（軽微）

#### 技術詳細
- 検出箇所: index.html:224
- 検出内容: `<img src="https://placehold.jp/150x150.png">`

#### 推奨対策
1. 実際のプロフィール画像を用意
2. 画像ファイルをプロジェクト内に配置
3. src属性を相対パスに変更

#### 対応工数目安
- 工数: 0.5時間
- 難易度: 低

### R-003: OGP・メタデータの未設定

| 項目 | 内容 |
|------|------|
| カテゴリ | 保守性 |
| 深刻度 | Low |
| 対応優先度 | 3（任意） |
| ステータス | 未対応 |

#### 概要
SNSでシェアされた際に表示される画像・説明文（OGP: Open Graph Protocol）が設定されていません。

#### ビジネスへの影響
- 影響範囲: SNSでのシェア時
- 想定被害: シェア時の見栄えが悪い（軽微）

#### 技術詳細
- 検出箇所: index.html `<head>` セクション
- 検出内容: og:title, og:description, og:image 等が未設定

#### 推奨対策
1. OGPメタタグを追加
2. シェア用画像（1200x630px推奨）を用意
3. 必要に応じてTwitterカードも設定

#### 対応工数目安
- 工数: 1時間
- 難易度: 低
