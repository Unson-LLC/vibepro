## リスク詳細

### R-001: データベース行レベルセキュリティ未設定

| 項目 | 内容 |
|------|------|
| **深刻度** | Critical |
| **カテゴリ** | セキュリティ |
| **影響範囲** | 全ユーザーのデータ |
| **検出ファイル** | supabase/migrations/001_initial.sql |

**リスク説明**:
データベースの行レベルセキュリティ（RLS）ポリシーが設定されていません。現在、データアクセス制御はアプリケーション層のみで行われており、データベースへの直接アクセスやサービスロールキーの漏洩時に、全ユーザーのデータが参照・変更可能な状態です。

**ビジネスへの影響**:
- 個人情報・ユーザーデータの漏洩リスク
- 不正なデータ変更・削除のリスク
- セキュリティインシデント発生時の信頼失墜

**推奨対応**:
```sql
ALTER TABLE "todo" ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Users can only access their own todos"
ON "todo" FOR ALL
USING (user_id = current_setting('request.jwt.claim.sub', true))
WITH CHECK (user_id = current_setting('request.jwt.claim.sub', true));
```

**対応工数**: 2〜4時間

### R-002: フレームワーク脆弱性（Next.js DoS）

| 項目 | 内容 |
|------|------|
| **深刻度** | High |
| **カテゴリ** | 依存関係 |
| **影響範囲** | サービス可用性 |
| **検出ファイル** | package.json |
| **CVE** | CVE-2026-23864 |

**リスク説明**:
使用している Next.js バージョン（16.1.4）に、HTTP リクエストのデシリアライズに関する脆弱性が存在します。特別に細工されたリクエストにより、サーバーのCPU使用率が急増、またはメモリ不足でクラッシュする可能性があります。

**ビジネスへの影響**:
- サービス停止（DoS 攻撃による）
- ユーザー体験の悪化
- 運用コストの増加（復旧対応）

**推奨対応**:
```bash
pnpm update next@16.1.5
```

**対応工数**: 1時間

### R-003: フレームワーク脆弱性（Image Optimizer DoS）

| 項目 | 内容 |
|------|------|
| **深刻度** | Medium |
| **カテゴリ** | 依存関係 |
| **影響範囲** | サービス可用性 |
| **検出ファイル** | package.json |
| **CVE** | CVE-2025-59471 |

**リスク説明**:
Next.js の Image Optimizer に、メモリ枯渇を引き起こす脆弱性があります。`remotePatterns` が設定されている場合、外部の大きな画像を処理させることでサーバーをクラッシュさせることが可能です。

**ビジネスへの影響**:
- 画像最適化機能使用時のサービス停止リスク
- 外部画像を使用する場合のセキュリティリスク

**推奨対応**:
```bash
pnpm update next@16.1.5
```

**対応工数**: 1時間（R-002 と同時対応）

**備考**: 現在の構成では `remotePatterns` 未使用のため、影響は限定的

### R-004: フレームワーク脆弱性（PPR Resume DoS）

| 項目 | 内容 |
|------|------|
| **深刻度** | Medium |
| **カテゴリ** | 依存関係 |
| **影響範囲** | サービス可用性 |
| **検出ファイル** | package.json |
| **CVE** | CVE-2025-59472 |

**リスク説明**:
Next.js の Partial Prerendering（PPR）機能に、メモリ枯渇を引き起こす脆弱性があります。PPR が有効な場合、特別なリクエストによりサーバーをクラッシュさせることが可能です。

**ビジネスへの影響**:
- PPR 機能使用時のサービス停止リスク

**推奨対応**:
```bash
pnpm update next@16.1.5
```

**対応工数**: 1時間（R-002 と同時対応）

**備考**: 現在の構成では PPR 未使用のため、影響は限定的

### R-005: セキュリティヘッダー未設定

| 項目 | 内容 |
|------|------|
| **深刻度** | Medium |
| **カテゴリ** | 設定 |
| **影響範囲** | クライアントセキュリティ |
| **検出ファイル** | next.config.ts |

**リスク説明**:
HTTP レスポンスにセキュリティ関連のヘッダー（X-Frame-Options、X-Content-Type-Options、Referrer-Policy 等）が設定されていません。これにより、クリックジャッキングやMIMEスニッフィング攻撃のリスクがあります。

**ビジネスへの影響**:
- クリックジャッキング攻撃によるフィッシングリスク
- セキュリティ監査での指摘事項

**推奨対応**:
```typescript
// next.config.ts
const nextConfig: NextConfig = {
  headers: async () => [
    {
      source: "/(.*)",
      headers: [
        { key: "X-Frame-Options", value: "DENY" },
        { key: "X-Content-Type-Options", value: "nosniff" },
        { key: "Referrer-Policy", value: "strict-origin-when-cross-origin" },
      ],
    },
  ],
};
```

**対応工数**: 1時間

### R-006: 認証ミドルウェアの検証不足

| 項目 | 内容 |
|------|------|
| **深刻度** | Low |
| **カテゴリ** | 認証 |
| **影響範囲** | ユーザー体験 |
| **検出ファイル** | src/middleware.ts |

**リスク説明**:
認証ミドルウェアでは Cookie の存在のみをチェックしており、セッションの有効性は検証していません。期限切れのセッションを持つユーザーが保護されたページにアクセスできる可能性があります（ただし、API 呼び出し時には正しく拒否されます）。

**ビジネスへの影響**:
- ユーザー体験の軽微な悪化（ページ表示後にエラー）
- セキュリティ上の実質的な影響は軽微

**推奨対応**:
1. 現状を許容する（Server Actions で認証チェック済み）
2. または、Middleware でセッション有効性を検証

**対応工数**: 2〜4時間
