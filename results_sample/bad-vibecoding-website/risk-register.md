## リスク詳細

### R-001: API キー・認証情報の公開

| 項目 | 内容 |
|------|------|
| カテゴリ | セキュリティ |
| 深刻度 | Critical |
| 対応優先度 | 1（即時対応必須） |
| ステータス | 未対応 |

#### 概要
JavaScript ファイル内に複数の API キーやアクセストークンがハードコードされており、サイト公開時にこれらの認証情報が外部に露出します。

#### ビジネスへの影響
- 影響範囲: サービス全体および連携サービス
- 想定被害:
  - OpenAI API の不正利用による高額課金
  - AWS リソースへの不正アクセス
  - Stripe 決済情報の漏洩
  - GitHub リポジトリへの不正アクセス
  - Slack チャンネルへの不正投稿

#### 技術詳細
- 検出箇所: `script.js:6-9`
- 検出内容:
  - OpenAI API Key (`sk-proj-...`)
  - Google API Key (`AIzaSy...`)
  - GitHub Personal Access Token (`ghp_...`)
  - Slack Webhook URL

#### 推奨対策
1. 該当コードを即時削除
2. 漏洩した可能性のあるキーをすべて無効化・再発行
3. API キーはバックエンドサーバーで管理
4. フロントエンドには公開可能な情報のみを含める

#### 対応工数目安
- 工数: 2-4時間
- 難易度: 低

### R-002: 認証情報ファイルの公開

| 項目 | 内容 |
|------|------|
| カテゴリ | セキュリティ |
| 深刻度 | Critical |
| 対応優先度 | 1（即時対応必須） |
| ステータス | 未対応 |

#### 概要
環境変数ファイル（.env）やサービスアカウントキーファイルが配信対象に含まれており、サイト公開時にこれらの機密情報が外部からアクセス可能になります。

#### ビジネスへの影響
- 影響範囲: サービス全体および連携サービス
- 想定被害:
  - データベースへの不正アクセス
  - クラウドサービス（AWS、GCP）への不正アクセス
  - 全システムへの侵入

#### 技術詳細
- 検出箇所:
  - `.env` - 複数の API キー、データベース接続情報、AWS 認証情報
  - `.env.local` - ローカル秘密情報
  - `config.secret.js` - API キー、データベース認証情報
  - `credentials.json` - GCP サービスアカウント秘密鍵

#### 推奨対策
1. これらのファイルを配信対象から除外
2. `.gitignore` に追加して Git 管理対象外に
3. 漏洩した可能性のあるキーをすべて無効化・再発行
4. 環境変数はサーバー側で管理

#### 対応工数目安
- 工数: 1-2時間
- 難易度: 低

### R-003: 悪意あるスクリプト実行の脆弱性（XSS）

| 項目 | 内容 |
|------|------|
| カテゴリ | セキュリティ |
| 深刻度 | Critical |
| 対応優先度 | 1（商用化前に対応必須） |
| ステータス | 未対応 |

#### 概要
URL パラメータやハッシュ値がサニタイズ（無害化）されずに画面に表示される処理があり、攻撃者が細工した URL をユーザーに踏ませることで、悪意あるスクリプトを実行させることができます。

#### ビジネスへの影響
- 影響範囲: サイト訪問者全員
- 想定被害:
  - ユーザーのセッション乗っ取り
  - フィッシング（偽の入力フォーム表示）
  - マルウェアのダウンロード誘導
  - 個人情報の窃取

#### 技術詳細
- 検出箇所:
  - `script.js:28` - URLパラメータ `name` を innerHTML に挿入
  - `script.js:35` - URLパラメータ `message` を innerHTML に挿入
  - `script.js:60` - location.hash を innerHTML に挿入
  - `script.js:42` - `eval()` の使用
  - `script.js:48` - `new Function()` の使用
  - `script.js:53` - `document.write()` の使用

#### 推奨対策
1. `innerHTML` を `textContent` に変更（HTML タグを解釈させない）
2. HTML を動的に生成する必要がある場合は DOMPurify でサニタイズ
3. `eval()`, `new Function()`, `document.write()` の使用を削除
4. Content Security Policy (CSP) ヘッダーの設定

#### 対応工数目安
- 工数: 4-8時間
- 難易度: 中

### R-004: マルウェア配信歴のある CDN 使用

| 項目 | 内容 |
|------|------|
| カテゴリ | セキュリティ |
| 深刻度 | Critical |
| 対応優先度 | 1（即時対応必須） |
| ステータス | 未対応 |

#### 概要
polyfill.io からスクリプトを読み込んでいます。このサービスは 2024 年に悪意ある第三者に売却され、実際にマルウェア配信に使用された実績があります。

#### ビジネスへの影響
- 影響範囲: サイト訪問者全員
- 想定被害:
  - マルウェア感染
  - 個人情報の窃取
  - サイトの信頼性低下
  - 法的責任のリスク

#### 技術詳細
- 検出箇所:
  - `index.html:10` - `https://polyfill.io/v3/polyfill.min.js`
  - `index.html:11` - `https://cdn.polyfill.io/v2/polyfill.min.js`

#### 推奨対策
1. polyfill.io からの読み込みを即時削除
2. 必要な場合は cdnjs.cloudflare.com を使用するか自己ホスティング
3. 現代のブラウザでは Polyfill が不要な場合も多いため、必要性を再検討

#### 対応工数目安
- 工数: 30分-1時間
- 難易度: 低

### R-005: デバッグログでの機密情報出力

| 項目 | 内容 |
|------|------|
| カテゴリ | セキュリティ |
| 深刻度 | High |
| 対応優先度 | 2（商用化前に対応必須） |
| ステータス | 未対応 |

#### 概要
開発用のデバッグログが残っており、API キーやアクセストークンがブラウザのコンソールに出力されます。

#### ビジネスへの影響
- 影響範囲: 開発者ツールを開いたユーザー
- 想定被害:
  - 認証情報の漏洩
  - API の不正利用

#### 技術詳細
- 検出箇所:
  - `script.js:12` - `console.log("API Key initialized:", OPENAI_API_KEY)`
  - `script.js:13` - `console.log("Auth token:", access_token)`

#### 推奨対策
1. 本番環境向けビルドからデバッグログを削除
2. 機密情報をログに出力しない設計に変更

#### 対応工数目安
- 工数: 30分
- 難易度: 低

| 項目 | 内容 |

2019 年にサービスを終了した rawgit.com からスクリプトを読み込んでいます。現在このサービスは機能しておらず、サイトの機能が正常に動作しない可能性があります。

- 影響範囲: 該当スクリプトに依存する機能
- 想定被害:
  - 機能の動作不良
  - JavaScript エラーによる UX 低下

- 検出箇所: `index.html:18` - `https://rawgit.com/user/repo/master/script.js`

#### 推奨対策
1. 該当スクリプトの読み込みを削除
2. 必要な機能は jsdelivr 等の CDN または自己ホスティングで提供

#### 対応工数目安
- 工数: 1-2時間

### R-007: 外部リソースの改ざんリスク

| 項目 | 内容 |
|------|------|
| カテゴリ | セキュリティ |
| 深刻度 | Medium |
| 対応優先度 | 3（商用化後でも可） |
| ステータス | 未対応 |

#### 概要
外部 CDN から読み込んでいる JavaScript / CSS に SRI（Subresource Integrity）が設定されていません。CDN が侵害された場合、改ざんされたファイルがサイト訪問者に配信される可能性があります。

#### ビジネスへの影響
- 影響範囲: サイト訪問者全員（CDN 侵害時）
- 想定被害:
  - マルウェア配信
  - サプライチェーン攻撃

#### 技術詳細
- 検出箇所:
  - `index.html:14` - jQuery（SRI なし）
  - `index.html:15` - Bootstrap CSS（SRI なし）

#### 推奨対策
1. 外部リソースに SRI ハッシュを追加
2. 可能であればセルフホスティングに移行

#### 対応工数目安
- 工数: 1-2時間
- 難易度: 低

### R-008: 外部埋め込みコンテンツのセキュリティ

| 項目 | 内容 |
|------|------|
| カテゴリ | セキュリティ |
| 深刻度 | Medium |
| 対応優先度 | 3（商用化後でも可） |
| ステータス | 未対応 |

#### 概要
YouTube、Vimeo、Google Maps の iframe 埋め込みに sandbox 属性が設定されていません。埋め込みコンテンツが侵害された場合のリスクを軽減できません。

#### ビジネスへの影響
- 影響範囲: 該当セクションを閲覧するユーザー
- 想定被害:
  - 埋め込みコンテンツからの攻撃（可能性は低い）

#### 技術詳細
- 検出箇所:
  - `index.html:342` - YouTube iframe（sandbox なし）
  - `index.html:343` - Vimeo iframe（sandbox なし）
  - `index.html:344` - Google Maps iframe（sandbox なし）

#### 推奨対策
1. iframe に適切な sandbox 属性を追加
2. 必要最小限の権限のみを許可

#### 対応工数目安
- 工数: 30分-1時間
- 難易度: 低

### R-009: 配信不要ファイルの混入

| 項目 | 内容 |
|------|------|
| カテゴリ | 保守性 |
| 深刻度 | Low |
| 対応優先度 | 4（任意） |
| ステータス | 未対応 |

#### 概要
本番環境に配信する必要のないファイル（node_modules、package.json、Git 関連ファイル等）が含まれています。これらは直接的なセキュリティリスクではありませんが、不要な情報の露出や配信サイズの増大につながります。

#### ビジネスへの影響
- 影響範囲: 配信効率・保守性
- 想定被害:
  - 不要な情報の露出
  - 配信サイズの増大
  - 保守性の低下

#### 技術詳細
- 検出箇所:
  - `node_modules/` - 開発用ディレクトリ
  - `package.json`, `pnpm-lock.yaml` - パッケージ管理ファイル
  - `.git/`, `.gitignore` - バージョン管理ファイル
  - `.wrangler/` - デプロイツール設定

#### 推奨対策
1. 配信時にこれらのファイルを除外
2. ビルドスクリプトで配信対象のみをコピー

#### 対応工数目安
- 工数: 30分
- 難易度: 低
