# 深刻度判定ガイド

診断結果からリスクの深刻度を判定する際の基準。

## 深刻度マトリクス

| 深刻度 | 発生可能性 | ビジネス影響 | 対応期限 |
|--------|------------|--------------|----------|
| Critical | 高（容易に悪用可能） | 甚大（情報漏洩・全面停止） | 即時 |
| High | 中〜高 | 大（重要機能停止・一部漏洩） | 商用化前 |
| Medium | 低〜中 | 中（機能劣化・UX低下） | 商用化後3ヶ月以内 |
| Low | 低 | 小（軽微な問題） | 任意 |

## カテゴリ別判定基準

### セキュリティ

| 検出内容 | 深刻度 | 根拠 |
|----------|--------|------|
| 秘密情報（APIキー等）の混入 | **Critical** | 即座に悪用可能、情報漏洩に直結 |
| 認証・認可の欠陥 | **Critical** | 不正アクセスに直結 |
| SQLインジェクション | **Critical** | データ漏洩・改ざんに直結 |
| XSS（ユーザ入力→innerHTML） | **High** | セッションハイジャックの可能性 |
| XSS（固定値→innerHTML） | **Low** | 悪用困難 |
| npm脆弱性（critical） | **High** | 既知の攻撃手法が存在 |
| npm脆弱性（high） | **High** | 既知の攻撃手法が存在 |
| npm脆弱性（medium） | **Medium** | 限定的な条件で悪用可能 |
| npm脆弱性（low） | **Low** | 悪用困難または影響軽微 |
| 外部CDN依存（SRIなし） | **Medium** | サプライチェーン攻撃のリスク |
| 外部CDN依存（SRIあり） | **Low** | 改ざん検知可能 |

### 可用性

| 検出内容 | 深刻度 | 根拠 |
|----------|--------|------|
| 単一障害点（SPOF） | **High** | サービス全面停止のリスク |
| DoS脆弱性 | **Medium**〜**High** | 攻撃難易度による |
| 外部サービス依存（フォールバックなし） | **Medium** | 外部障害時に影響 |
| オートスケール未設定 | **Medium** | 負荷増大時に停止リスク |

### 技術的負債

| 検出内容 | 深刻度 | 根拠 |
|----------|--------|------|
| 非推奨APIの使用 | **Medium** | 将来的に動作しなくなる可能性 |
| 古いライブラリバージョン | **Low**〜**Medium** | セキュリティリスクの有無による |
| テストカバレッジ不足 | **Medium** | 変更時の品質担保が困難 |
| ドキュメント不足 | **Low** | 保守性への影響 |

### 運用

| 検出内容 | 深刻度 | 根拠 |
|----------|--------|------|
| 監視未設定 | **High** | 障害検知の遅延 |
| ログ出力不足 | **Medium** | 障害調査が困難 |
| バックアップ未設定 | **High** | データ損失のリスク |
| 復旧手順未整備 | **Medium** | 障害復旧の遅延 |

## 判定フローチャート

```
検出された問題
    │
    ▼
[悪用された場合の影響は？]
    │
    ├─ 情報漏洩・全面停止 → Critical
    │
    ├─ 重要機能停止・一部影響 → High
    │
    ├─ 機能劣化・UX低下 → Medium
    │
    └─ 軽微・間接的 → Low

    │
    ▼
[悪用の容易さは？]
    │
    ├─ 容易（公開情報で攻撃可能） → 1段階上げ
    │
    ├─ 困難（特殊条件が必要） → 1段階下げ
    │
    └─ 通常 → 変更なし
```

## 深刻度の調整

以下の条件で深刻度を調整:

### 上方修正（より深刻に）
- インターネット公開サービスである
- 個人情報・決済情報を扱う
- ユーザー数が多い（1,000人以上）
- 攻撃手法が公開されている

### 下方修正（より軽微に）
- 社内ツール（限定公開）
- 機密データを扱わない
- 悪用に特殊な条件が必要
- 既に緩和策が存在する
