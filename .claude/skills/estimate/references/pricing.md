## 規模判定ガイド

### 判定基準表

| 判定項目 | ライト | スタンダード | エンタープライズ |
|----------|--------|--------------|------------------|
| Critical/High リスク数 | 0-2件 | 3-5件 | 6件以上 |
| 総リスク数 | 0-5件 | 6-15件 | 16件以上 |
| 外部連携 | なし | 1-3サービス | 4サービス以上 |
| 認証機能 | なし/簡易 | 標準 | SSO/多要素 |

### 判定フロー

```text
Critical リスクあり?
  → Yes → エンタープライズ検討
  → No → High リスク数を確認

High リスク 3件以上?
  → Yes → スタンダード以上
  → No → 総リスク数を確認

総リスク 15件以上?
  → Yes → スタンダード
  → No → ライト
```

### 静的サイト向け簡易判定

静的サイト（HTML/CSS/JS のみ、サーバーサイド処理なし）の場合：

| 条件 | 判定規模 |
|------|----------|
| リスクなし or Low のみ | ライト |
| Medium あり、Critical/High なし | ライト |
| High あり、Critical なし | スタンダード |
| Critical あり | スタンダード以上（要相談） |

### 追加考慮事項

以下の場合は規模を1段階上げることを検討：

- 大規模な JavaScript アプリケーション（React/Vue/Angular等）
- 多数の外部 API 連携（3サービス以上）
- 認証・決済機能を含む
- 個人情報を扱う
