# 秘密情報混入チェック

秘密情報（APIキー/トークン/パスワード等）の混入を検出・報告する。

## 検出パターン

```
# APIキー/トークンパターン
(api[_-]?key|api[_-]?secret|access[_-]?token|auth[_-]?token)
sk-[a-zA-Z0-9]{20,}
[a-z0-9]{32,}

# 認証情報ファイル
\.env$|\.env\.|credentials|\.key$|\.pem$
```

## リスク

- APIキー漏洩による不正利用・課金
- 認証情報漏洩によるシステム侵害
- Git履歴に残ると削除が困難

## チェック項目

### 1. 認証情報ファイル

検出対象:
- `.env` / `.env.*`
- `credentials.json` / `credentials.yml`
- `*.key` / `*.pem` / `*.p12`
- `secrets.json` / `config.secret.js`

報告内容:
- ファイル名とパス
- ファイルの種類（環境変数/証明書/設定ファイル）
- 公開される設定にいる場合は警告をする

### 2. コード内のAPIキー

検出対象:
- ハードコードされたAPIキー
- トークン文字列（sk-xxx, Bearer xxx など）
- 長いランダム文字列（32文字以上）

報告内容:
- 検出箇所（ファイル名:行番号）
- キーの種類（推定）
- 公開可能かどうかの判断材料

### 3. デバッグログ内の機密情報

検出対象:
- `console.log` で出力されるレスポンス全体
- 認証ヘッダーのログ出力
- トークンを含む変数のログ出力

報告内容:
- 検出箇所（ファイル名:行番号）
- 出力される可能性のある情報

## チェック結果の記載例

```markdown
## 秘密情報混入

### 検出結果
- 認証情報ファイル: 1件
- コード内APIキー: 2件
- デバッグログ: 1件

### 認証情報ファイル

| ファイル | 種類 |
|----------|------|
| .env.local | 環境変数ファイル |

### コード内APIキー

| ファイル | 行 | 検出内容 | 種類（推定） |
|----------|-----|----------|--------------|
| api.js | 15 | sk-xxxx... | OpenAI API Key |
| config.js | 8 | AIzaSy... | Google API Key |

### デバッグログ

| ファイル | 行 | 内容 |
|----------|-----|------|
| auth.js | 42 | console.log(response) - レスポンス全体を出力 |
```
