# コード品質診断

`target/` ディレクトリ内のコードに対してコード品質診断を実行し、結果を `results/code-quality.md` に出力してください。

## 診断観点

### 1. 型安全性
- TypeScript使用時の `any` 型の乱用
- 型定義の不足
- `@ts-ignore` / `@ts-nocheck` の使用
- 暗黙的な型変換

### 2. エラーハンドリング
- try-catchの適切な使用
- エラーの握りつぶし（空のcatchブロック）
- 非同期処理のエラーハンドリング
- ユーザーフレンドリーなエラーメッセージ

### 3. コード構造
- 関数・クラスの単一責任原則
- 関数の長さ（50行以上は警告）
- ネストの深さ（4段以上は警告）
- 重複コード（DRY原則違反）

### 4. テストカバレッジ
- テストファイルの有無
- 主要機能のテストカバレッジ
- エッジケースのテスト
- モックの適切な使用

### 5. コーディング規約
- 一貫した命名規則
- デッドコード（未使用の変数・関数）
- console.log の残存
- TODO/FIXMEコメントの放置

### 6. 依存関係管理
- 未使用の依存関係
- 開発依存と本番依存の分離
- バージョン固定（lockファイル）

## 出力形式

以下の形式で `results/code-quality.md` に出力してください：

```markdown

診断日時: YYYY-MM-DD HH:MM
対象: target/

## サマリー

| カテゴリ | スコア | 評価 |
|----------|--------|------|
| 型安全性 | X/10 | A〜D |
| エラーハンドリング | X/10 | A〜D |
| コード構造 | X/10 | A〜D |
| テスト | X/10 | A〜D |
| コーディング規約 | X/10 | A〜D |
| **総合** | **X/10** | **A〜D** |

評価基準: A(8-10), B(6-7), C(4-5), D(0-3)

## 検出項目

### 型安全性

#### [CQ-001] 項目タイトル
- **ファイル**: `path/to/file.ts:行番号`
- **重要度**: High/Medium/Low
- **内容**: 問題の説明
- **推奨対応**: 修正方法

### エラーハンドリング
...

### コード構造
...

### テスト
...

### コーディング規約
...

## 良い点
- 適切に実装されている点があればリストアップ

## 改善優先度
1. 最優先で改善すべき項目
2. 次に改善すべき項目
3. 余裕があれば改善する項目
```

## 実行手順

1. `target/` ディレクトリ内の全ファイルを探索
2. 各診断観点についてコードを分析
3. カテゴリごとにスコアリング
4. `results/code-quality.md` に結果を出力
5. 完了メッセージを表示
