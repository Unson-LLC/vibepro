# アーキテクチャ診断

`target/` ディレクトリ内のコードに対してアーキテクチャ診断を実行し、結果を `results/architecture.md` に出力してください。

## 診断観点

### 1. プロジェクト構造
- ディレクトリ構成の適切性
- 関心の分離（Separation of Concerns）
- レイヤードアーキテクチャの遵守
- モジュール間の依存方向

### 2. 技術スタック評価
- フレームワーク・ライブラリの選定
- バージョンの新しさ・サポート状況
- 技術的負債となりうる選択
- 代替技術の検討

### 3. スケーラビリティ
- 水平スケーリングの可能性
- ステートレス設計
- データベース設計（N+1問題等）
- キャッシュ戦略

### 4. 依存関係
- 外部サービスへの依存度
- サードパーティAPIの障害対策
- 循環依存の検出
- 疎結合の実現度

### 5. データフロー
- データの流れの明確さ
- 状態管理の複雑さ
- API設計の一貫性（REST/GraphQL）
- データ整合性の担保

### 6. セキュリティアーキテクチャ
- 認証・認可の設計
- APIゲートウェイの有無
- ネットワーク境界の設計

## 実行手順

1. `target/` ディレクトリ内の構造を分析
2. package.json等から技術スタックを特定
3. 各診断観点について評価
4. `results/architecture.md` に結果を出力
5. 完了メッセージを表示
