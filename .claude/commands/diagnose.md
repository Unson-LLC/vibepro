# 総合診断（フェーズA診断）

`target/` ディレクトリ内のコードに対して総合診断を実行します。

このコマンドは以下の個別診断を順番に実行し、最終的に統合サマリーを生成します：

1. `/diagnose-security` → `results/security.md`
2. `/diagnose-code-quality` → `results/code-quality.md`
3. `/diagnose-architecture` → `results/architecture.md`
4. `/diagnose-operations` → `results/operations.md`
5. `/diagnose-scale` → `results/scale.md`
6. 統合サマリー生成 → `results/summary.md`

## 実行手順

### Step 1: 個別診断の実行

各診断コマンドの指示に従い、以下のファイルを生成してください：

1. **セキュリティ診断**: `.claude/commands/diagnose-security.md` の指示に従い実行
2. **コード品質診断**: `.claude/commands/diagnose-code-quality.md` の指示に従い実行
3. **アーキテクチャ診断**: `.claude/commands/diagnose-architecture.md` の指示に従い実行
4. **運用準備度診断**: `.claude/commands/diagnose-operations.md` の指示に従い実行
5. **規模判定**: `.claude/commands/diagnose-scale.md` の指示に従い実行

### Step 2: 統合サマリーの生成

全ての個別診断が完了したら、`results/summary.md` に以下の形式で統合レポートを生成してください：

```markdown

診断日時: YYYY-MM-DD HH:MM
対象: target/

---

- [ ] **大幅な改修が必要**: 商用化には相当の作業が必要
```
