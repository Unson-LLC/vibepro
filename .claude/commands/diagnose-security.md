# セキュリティ診断

`target/` ディレクトリ内のコードに対してセキュリティ診断を実行し、結果を `results/security.md` に出力してください。

## 診断観点

### 1. 認証・認可
- 認証機構の有無と実装方法
- セッション管理の安全性
- 認可チェックの漏れ
- JWTの適切な実装（有効期限、署名検証等）

### 2. 入力バリデーション
- ユーザー入力のサニタイズ
- SQLインジェクション対策
- XSS（クロスサイトスクリプティング）対策
- パストラバーサル対策
- コマンドインジェクション対策

### 3. シークレット管理
- APIキーやパスワードのハードコード検出
- 環境変数での機密情報管理
- `.env` ファイルの `.gitignore` 設定

### 4. 依存関係の脆弱性
- 既知の脆弱性を持つパッケージの使用
- 依存パッケージのバージョン管理

### 5. データ保護
- 機密データの暗号化
- HTTPS強制
- CORS設定の適切性

## 出力形式

以下の形式で `results/security.md` に出力してください：

```markdown
# セキュリティ診断レポート

診断日時: YYYY-MM-DD HH:MM
対象: target/

## サマリー

| 重要度 | 件数 |
|--------|------|
| Critical | X件 |
| High | X件 |
| Medium | X件 |
| Low | X件 |

## 検出項目

### Critical

#### [SEC-001] 項目タイトル
- **ファイル**: `path/to/file.ts:行番号`
- **内容**: 問題の説明
- **リスク**: 想定されるリスク
- **推奨対応**: 修正方法

### High
...

### Medium
...

### Low
...

## 良い点
- 適切に実装されている点があればリストアップ

## 推奨事項
- 今後のセキュリティ強化のための提案
```

## 実行手順

1. `target/` ディレクトリ内の全ファイルを探索
2. 各診断観点についてコードを分析
3. 問題を重要度別に分類
4. `results/security.md` に結果を出力
5. 完了メッセージを表示
