# 運用準備度診断

`target/` ディレクトリ内のコードに対して運用準備度診断を実行し、結果を `results/operations.md` に出力してください。

## 診断観点

### 1. ログ出力
- 構造化ログの実装
- ログレベルの適切な使い分け（debug/info/warn/error）
- リクエストID/トレースIDの付与
- 機密情報のマスキング
- ログローテーション設定

### 2. 監視・可観測性
- ヘルスチェックエンドポイント
- メトリクス出力（Prometheus等）
- 分散トレーシング対応
- エラートラッキング（Sentry等）連携

### 3. 設定管理
- 環境変数による設定外部化
- 設定のバリデーション
- 環境別設定（dev/staging/prod）
- フィーチャーフラグ対応

### 4. デプロイ準備
- Dockerfile / コンテナ化対応
- CI/CD設定ファイルの有無
- データベースマイグレーション
- ロールバック戦略

### 5. ドキュメント
- README.mdの充実度
- API仕様書（OpenAPI等）
- 環境構築手順
- トラブルシューティングガイド

### 6. 障害対策
- グレースフルシャットダウン
- リトライ・サーキットブレーカー
- バックアップ戦略
- 障害時の切り分け手段

## 実行手順

1. `target/` ディレクトリ内のファイルを探索
2. 設定ファイル、Dockerfile、CI設定等を確認
3. 各診断観点についてチェック
4. `results/operations.md` に結果を出力
5. 完了メッセージを表示
