# Security Policy

## Supported Versions

VibePro is pre-1.0 software. Security fixes target the latest published version and `main`.

## Reporting a Vulnerability

Please do not open public issues for vulnerabilities.

Report suspected vulnerabilities by opening a private GitHub Security Advisory for this repository:

https://github.com/Unson-LLC/vibepro/security/advisories/new

If private advisories are unavailable, contact the maintainers through the security contact listed in the GitHub repository. Do not include exploit details, secrets, or private repository evidence in public issues.

Include:

- affected version or commit
- reproduction steps
- expected impact
- whether secrets, generated evidence, or repository contents may be exposed

## Data Handling

VibePro stores evidence under `.vibepro/` in the target repository. Users are responsible for excluding local evidence, logs, customer data, and secrets from public commits and package publications.
