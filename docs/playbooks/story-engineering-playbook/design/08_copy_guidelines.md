# コピー・文言ルール

UI コピーは、ユーザーに判断と操作を促すための仕様です。実装者の仮置き文言を
そのまま UI に残さず、Story の対象ユーザーと状態に合わせて決めます。

## 基本方針

- ボタンは実行される動作を表す
- 見出しは画面や領域の目的を表す
- エラーはユーザーが次にできる行動を含める
- 成功メッセージは完了した対象を明示する
- 内部実装名、ログ用語、例外名をユーザー向け文言に出さない

## 確認観点

- 対象ユーザーに伝わる業務語になっている
- 同じ操作に同じ言葉を使っている
- ラベル、ヘルプ、エラー、空状態の文体が揃っている
- 翻訳や日本語/英語の差し替えが必要な箇所を識別できる
