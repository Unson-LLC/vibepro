# 情報設計

UI/UX の情報設計は、画面を装飾する前に、ユーザーが何を判断し、どの順番で
操作するかを固定するための設計面です。

## 確認観点

- 主要ユーザーと利用シーンが [ペルソナ](../product/03_personas.md) に接続している
- 重要な判断材料が画面上で先に見える
- 作成、確認、修正、完了の流れが Story の受け入れ条件に対応している
- 空状態、読み込み中、エラー、成功後の遷移が分断されていない
- スマートフォンとデスクトップで情報の優先順位が変わらない

## 成果物

- 主要画面または主要領域の一覧
- 画面間の遷移または状態遷移
- ユーザーが判断に使う情報の優先順位
- Story / Spec / Architecture との対応
