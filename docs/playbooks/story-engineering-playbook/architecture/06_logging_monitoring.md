# ログ・監視設計

## 1. ログ方針

- 重要な操作は監査ログに残す
- エラーは追跡可能な形で残す
- 個人情報・秘密情報はログに出さない

## 2. ログレベル

| Level | 用途 |
|---|---|
| debug | 開発時の詳細確認 |
| info | 通常操作 |
| warn | 注意が必要な状態 |
| error | 障害・失敗 |

## 3. 記録してはいけない情報

- パスワード
- トークン
- クレジットカード情報
- 個人情報の詳細
- 秘密鍵
