# エラー設計

## 1. 基本方針

- ユーザーが次に何をすればよいか分かる文言にする
- 内部エラー詳細をユーザーに出さない
- APIエラーは共通形式で返す
- 権限エラーと認証エラーを区別する

## 2. 共通エラーレスポンス

```json
{
  "error": {
    "code": "ERROR_CODE",
    "message": "ユーザー向けメッセージ"
  }
}
```

## 3. HTTPステータスの使い分け

| Status | 用途 |
|---:|---|
| 400 | 不正なリクエスト |
| 401 | 未ログイン |
| 403 | 権限なし |
| 404 | 対象なし |
| 409 | 競合 |
| 500 | サーバーエラー |
