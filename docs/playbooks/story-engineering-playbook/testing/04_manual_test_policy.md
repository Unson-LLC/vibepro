# 手動確認方針

## 1. 手動確認の目的

手動確認は、自動テストでは確認しにくい体験品質を確認するために行います。

特に、UI/UX、文言、スマホ表示、現場での使いやすさを確認します。

## 2. 手動で確認するもの

- 文言が自然か
- 上から目線の表現になっていないか
- 画面の違和感がないか
- スマホ表示が崩れていないか
- ボタンが押しやすいか
- エラー時にユーザーが迷わないか
- ローディング中の挙動が自然か
- 現場で使いやすいか
- 顧客目線で不安がないか

## 3. 確認環境

| 環境 | 確認内容 |
|---|---|
| PC Chrome | 基本表示・操作 |
| PC Safari | 表示崩れ確認 |
| iPhone Safari | スマホ操作・表示 |
| Android Chrome | 必要に応じて確認 |
