## 12. 06_tasks.md に反映するテスト条件

各タスクには、以下の形式でテスト条件を入れます。

TDDで進める場合は、1つの振る舞いごとに `RED → GREEN → REFACTOR` を回します。
まとめて全テストを書いてからまとめて実装する進め方は避けます。

```md
## テスト条件

### 単体テスト
-

### 統合テスト
-

### E2Eテスト
-

### 手動確認
-

### 今回テストしないもの
-
```

## 13. 品質ゲート

このfeatureでPR前、Issue完了前、リリース前に確認する品質ゲートを書きます。

| 観点 | 確認内容 | 確認方法 | 対象外理由 |
|---|---|---|---|
| 正しさ | 受け入れ条件、例外、境界値、状態遷移 | 自動 / 手動 |  |
| 設計整合性 | 既存パターン、責務分離、ディレクトリ方針 | レビュー |  |
| セキュリティ | 入力検証、認証・権限、秘密情報、ログ | 自動 / レビュー |  |
| パフォーマンス | N+1、無制限取得、不要な再レンダー、アセット | 自動 / 手動 / レビュー |  |
| アクセシビリティ | キーボード操作、ラベル、フォーカス、コントラスト | E2E / 手動 / レビュー |  |
| 運用 | ログ、監視、ロールバック、feature flag | レビュー / 手動 |  |
