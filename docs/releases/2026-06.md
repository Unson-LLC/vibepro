# June 2026

**Type:** Development milestone + npm alpha/beta · **Merged PRs:** 115

## npm Publication

Versions `0.1.0-alpha.0` and `0.1.0-beta.0` were published to npm on June 7. `0.1.0-beta.0` remains the `latest` and `beta` tag.

## From Journey to a Managed Merge

[PR #133](https://github.com/Unson-LLC/vibepro/pull/133) added Patton-style Journey Maps, [PR #145](https://github.com/Unson-LLC/vibepro/pull/145) an isolated-worktree Execution DAG, [PR #149](https://github.com/Unson-LLC/vibepro/pull/149) an evidence-backed judgment spine, and [PR #157](https://github.com/Unson-LLC/vibepro/pull/157) the Business/Dev Story contract. Starting with [PR #163](https://github.com/Unson-LLC/vibepro/pull/163), merges became part of VibePro's audited path.

## Review and Senior Judgment

[PR #168](https://github.com/Unson-LLC/vibepro/pull/168) added review-dispatch preflight, [PR #174](https://github.com/Unson-LLC/vibepro/pull/174) judgment handoff fields, and [PR #184](https://github.com/Unson-LLC/vibepro/pull/184) a multi-axis senior-judgment DAG. [PR #190](https://github.com/Unson-LLC/vibepro/pull/190) allowed safe review reuse across merge deltas.

## Post-Merge Audit and Evidence Cost

[PR #200](https://github.com/Unson-LLC/vibepro/pull/200) introduced canonical audit artifacts, [PR #216](https://github.com/Unson-LLC/vibepro/pull/216) reused equivalent evidence, and [PR #224](https://github.com/Unson-LLC/vibepro/pull/224) produced a compressed replay package. Design SSOT reconciliation ([PR #234](https://github.com/Unson-LLC/vibepro/pull/234)), session-efficiency audit ([PR #240](https://github.com/Unson-LLC/vibepro/pull/240)), bundled codebase-memory support ([PR #244](https://github.com/Unson-LLC/vibepro/pull/244)), and architecture readiness ([PR #251](https://github.com/Unson-LLC/vibepro/pull/251)) completed the month.
