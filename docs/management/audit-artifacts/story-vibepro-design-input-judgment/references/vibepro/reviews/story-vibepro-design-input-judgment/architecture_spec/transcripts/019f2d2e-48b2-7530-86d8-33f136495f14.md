## Finding

`pr prepare` resolved design-input judgment evidence only through `latest_run_by_story`.
If a story first ran `story diagnose --pre-architecture` and later ran a normal
pre-implementation diagnosis, the later run became the latest run and hid the
earlier design-input evidence.

## Reproduction Shape

1. Run design-input diagnosis for the story.
2. Run pre-implementation diagnosis for the same story.
3. Run PR preparation.
4. Observe that design-input judgment is missing because PR context only reads
   the latest diagnosis run for the story.

## Required Fix

Preserve the design-input run independently of the later pre-implementation run,
and add regression coverage that verifies both judgments can coexist after the
later diagnosis.
