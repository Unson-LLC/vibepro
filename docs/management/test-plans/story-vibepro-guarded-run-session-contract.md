# Test strategy

The Run Session module is covered primarily by deterministic unit/integration tests using temporary repositories. CLI tests prove routing, rendering, exit codes, and exact legacy compatibility. No test may use a real managed worktree outside its temporary fixture.

## Path and surface matrix

| Surface | Run selection | Output | Invocation context | Expected evidence |
|---|---|---|---|---|
| `execute run` | generated | human + JSON | managed mode disabled + source control root + recorded managed root + new preferred/required unavailable + pre-existing unavailable binding | valid opaque id, explicit `repository`/`managed`/`source_fallback` authority, no nested/replacement worktree, matching preferred fallback, fail-closed existing binding, creation lock, authority-first commit, linked-copy synchronization, and creation partial failure with recoverable `run_id` |
| `execute status` | omitted | human + JSON | source + managed | byte-for-byte legacy output and no Run artifact mutation |
| `execute status` | explicit valid/invalid/missing | human + JSON | source + managed + missing authority | selected authoritative Run or typed exit-2 error; mirror is never fallback |
| `execute watch` | explicit + omitted | human + JSON | source + managed + missing authority | ordered authoritative snapshot; latest by `created_at`, then `run_id`; any rejected implicit candidate, drift, or unavailable authority is typed and visible |
| `execute watch --repair-linked-copy` | explicit + omitted | human + JSON | source + managed + repository/source-fallback | mirror restored from authority with no Run transition, or typed nonmutating `linked_copy_not_configured` |
| `execute resume` | explicit + omitted | human + JSON | recorded source control, managed authority, unmanaged-secondary, wrong, missing, stale-HEAD worktree | same Run and incremented attempt from either allowed managed control root, or nonmutating typed stop |
| `execute cancel` | explicit + omitted | human + JSON | source + managed + missing authority | cancelled transition; after recognized migration reaches canonical `0.1.0`, repeated cancel preserves artifact bytes and transition count |

## State and persistence cases

- Fresh `execute run` fixes one injected clock reading `t` and asserts the complete initial persisted contract: `schema_version=0.1.0`, generated `run_id`, `story_id`, `target=pr_ready`, `autonomy_mode=guarded`, `created_at=updated_at=last_progress_at=t`, `status=running`, `stop_reason=null`, `attempt=1`, `iteration=0`, `budget={max_attempts:1,max_iterations:0}`, `deadline=null`, `pending_decision=null`, binding-derived `current_head_sha`, a closed authority kind, full execution-context and managed-binding snapshots, and exactly `[{sequence:1,from:null,to:"running",reason:"run_created",timestamp:t}]`. Human and JSON CLI creation tests assert the same defaults. A boundary fixture resumes explicitly from `attempt=1` to `attempt=2` despite `max_attempts=1`, proving that budget is advisory in this persistence-only Story; no command increments `iteration`, and tests do not imply autonomous budget enforcement before the orchestrator Story.
- The only recognized predecessors are a Run missing `schema_version` and a Run with `schema_version: "0.0.0"`, each already satisfying every other `0.1.0` required field and validation rule. Migration adds or replaces only `schema_version`; fixtures use non-default identity/binding, `stop_reason`, `budget`, `deadline`, `last_progress_at`, `current_head_sha`, `pending_decision`, `updated_at`, and transitions and prove they are deep-preserved. Every other missing required field returns nonmutating `invalid_state`; there are no field defaults beyond the schema-version mapping. An older `cancelled` Run cancelled during that read performs migration only, without a new transition or `updated_at` change; its next cancel is byte-for-byte unchanged. A recognized older `source_fallback` fixture migrates only after valid authority/fingerprint checks, while missing or mismatched fingerprints return nonmutating `invalid_state` or `worktree_mismatch`.
- Recognized older managed state first compares raw authority and mirror artifact bytes. Pre-existing divergence returns `linked_copy_out_of_sync` with both old copies byte-for-byte unchanged. Matching copies then migrate authority first and mirror second to the same canonical bytes. Injected mirror failure returns `linked_copy_sync_failed` with `run_id` and both artifacts after exactly one authority migration; restart/read returns `linked_copy_out_of_sync`, explicit repair copies authority to mirror without a transition, and a later repeated cancel is byte-for-byte unchanged.
- Corrupt JSON is moved beside the authority to exactly `state.json.corrupt-<UTC timestamp>`, returns exit-2 `corrupt_state`, includes `quarantine_artifact` in JSON details and the artifact reference in human output, and writes no replacement valid state.
- A future schema returns `unsupported_schema` and remains byte-for-byte unchanged.
- Invalid state and unknown status return typed contract errors and preserve the last valid artifact. A table-driven test covers every status pair across `running`, `waiting_for_human`, `waiting_for_runtime`, `blocked`, `failed`, `cancelled`, and `pr_ready`: all Architecture-listed transitions pass, every other pair fails without mutation, `failed -> running` succeeds only through explicit resume, `pr_ready` is absorbing, and repeated canonical `0.1.0` `cancelled -> cancelled` is a byte-for-byte no-op.
- Multiple Runs use `created_at` descending and `run_id` descending as a deterministic tie-break; zero Runs returns `run_not_found`.
- Invalid Story and Run IDs, including raw traversal strings and encoded separators, fail before path composition.
- Every Run records canonical root and Git directory identity; a second unmanaged worktree at the same HEAD cannot resume it.
- Managed authority is selected from existing execution metadata before bootstrap. Fixtures cover source invocation, recorded managed invocation, newly attempted preferred-unavailable source fallback, newly attempted required-unavailable failure, and prohibit nested managed worktree creation.
- Pre-existing legacy metadata with `managed_worktree.status=unavailable` fails `execute run` with `worktree_unavailable` from the source root and, when its recorded path still resolves, from the recorded managed root. A physically missing recorded authority has the same source-root result. Each fixture proves no source fallback, `startExecution`, replacement/nested worktree, Run artifact, or mirror promotion occurs in either `preferred` or `required` mode.
- Managed-worktree `mode=disabled` creates a `repository` Run bound to the canonical source root/Git directory/HEAD with no linked mirror. After restart, status/watch/resume/cancel work only from that root, and repair returns exit-2 `linked_copy_not_configured` while preserving the authority bytes, `updated_at`, and transitions exactly.
- A newly attempted preferred-mode unavailable bootstrap records a source-authoritative Run with `authority_kind=source_fallback` and a SHA-256 `bootstrap_binding_fingerprint` over the fixed canonical tuple `status`, `mode`, canonical `source_repo`, `relative_path`, expected `branch`, `actual_branch`, `base_ref`, `created_from_sha`, `current_head_sha`, and `failure_reason` returned and persisted by `startExecution` (absent optional values normalize to `null`; fixed key order). The fixture proves derived fields need not be duplicated and no legacy `execution_id` is required. After a simulated process restart, explicit status and implicit/explicit watch, resume, and cancel all recover that same Run from the recorded source root despite the matched unavailable legacy metadata; repair returns exit-2 `linked_copy_not_configured` without byte or transition mutation. Missing fingerprints, every one-field fingerprint mismatch including branch-only mismatch, and unknown `authority_kind` values fail without treating an arbitrary local artifact as authority.
- Concurrent Run creation uses an exclusive Story lock: one attempt proceeds and the other returns nonmutating `run_creation_locked`. A pre-existing orphan-lock fixture proves exit-2 human/JSON output includes the lock artifact, performs no lock stealing, bootstrap, legacy/Run/mirror mutation, or cleanup of the orphan. A normal bootstrap exception proves the current invocation removes its own lock in `finally` and a subsequent invocation can proceed.
- A failure-injection fixture makes managed creation return unavailable and the subsequent legacy linked-copy write throw after the source legacy state commits. `execute run` returns exit-2 `legacy_bootstrap_partial` with the legacy artifact and underlying failure details, preserves that legacy state, creates no Run or mirror, and releases its creation lock. A second `execute run` sees the pre-existing unavailable binding and returns `worktree_unavailable`; no path promotes the partial state to source fallback. If no legacy artifact was committed, a CLI fixture proves the original unexpected failure exits `1`, uses the internal-error path rather than the contract envelope, and creates no Run.
- Source and managed callers bind `execution_context` to the same authoritative root/Git directory/HEAD; another unmanaged worktree at the same HEAD is not an allowed control root.
- Deleting a recorded managed authority makes status/watch/resume/cancel/repair return `worktree_unavailable` in human and JSON form without reading the mirror as authority or mutating either copy.
- Injected mirror failure proves an existing-Run authority transition commits once and retry does not repeat it. Creation failure returns the committed `run_id`; rerunning `execute run` intentionally creates another Run, while repairing the returned Run changes only its mirror.

## CLI UX and restart matrix

- `vibepro execute --help` and each supported `execute <run|status|watch|resume|cancel> --help` route expose the additive commands, `--run-id`, `--repair-linked-copy`, `target=pr_ready`, `autonomy=guarded`, and the explicit-Run versus implicit-legacy `status` distinction while preserving all legacy execute entries.
- Each explicit Run command tests valid, syntactically invalid, and well-formed missing `run_id`. `watch`, `resume`, and `cancel` additionally test omitted selection with zero, one, multiple, and equal-`created_at` Runs. A mixed valid/invalid candidate fixture proves implicit selection returns nonmutating `run_selection_blocked` with rejected Run ids, reason codes, artifacts, and explicit-selection guidance instead of silently targeting an older Run. `status` without `--run-id` is tested only through the exact legacy route. Repair is exercised through explicit and omitted watch selection.
- For explicit `status`, explicit/implicit `watch`, `resume`, `cancel`, and repair where applicable, a table covers recorded source control root, managed authority root, wrong checkout, an unmanaged same-HEAD worktree, and deleted/missing authority. Accepted roots resolve one authority; rejected roots and missing authorities return typed nonmutation.
- A subprocess-based fresh-process fixture reloads a managed Run from both allowed control roots and exercises status, explicit/implicit watch, resume, cancel, and repair. Repository and source-fallback restart fixtures remain separate and prove their narrower root/mirror behavior.
- Successful human output asserts Run id, status, stop reason, binding, and ordered transitions. Successful JSON is deep-equal to the persisted Run and contains no presentation-only fields.
- Representative validation, selection, authority, transition, lock, corruption, and linked-copy failures assert exit `2` plus the exact JSON envelope `{status:"error",stop_reason:{code,message,details}}`; human output asserts code, message, and any applicable artifact reference. The injected unexpected failure fixture asserts exit `1`.

## Authority regressions

- Factory injection provides deterministic `now`, `randomBytes`, `startExecution`, Gate-readiness, minimal artifact IO including `readdir`, and Git-identity dependencies; it rejects unknown keys. `runCli` accepts only `io.guardedRunDependencies` for in-process CLI failure injection, passes them through that factory, and has no whole-service replacement seam. Tests never monkeypatch immutable ESM imports.
- A static import/call-surface assertion proves `guarded-run-session.js` and the `runCli` seam neither import nor accept agent/runtime dispatch, arbitrary-action/shell dispatch, waiver, merge, or whole-service replacement capabilities.
- A non-ready Gate DAG fixture proves the internal transition boundary cannot persist `pr_ready` and leaves authority/mirror unchanged; a ready Gate DAG fixture is the only positive `pr_ready` path and records one validated transition.
- `startExecution` and Gate-readiness spies plus before/after artifacts prove Run commands cannot mutate legacy execution state except for the single bootstrap invoked by `execute run`; status/watch/resume/cancel/repair never call bootstrap, and the static surface assertion closes waiver/merge authority.
- Existing `execute start/status/next/reconcile/merge` behavior remains unchanged, including intentional mutations by start/reconcile/merge; status output remains byte-for-byte compatible and status is nonmutating.
