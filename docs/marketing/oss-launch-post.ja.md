# VibePro OSS公開 告知草案

## 中心メッセージ

AIを見張り続けるのをやめる。作りたかったものを出荷する。

VibePro は、Codex や Claude Code を日常的に使っている開発者向けのツールです。AIに実装を任せたい。でも、毎回横で見張り、レビューのたびに意図を説明し直し、PRが本当に出せる状態かを勘で判断したくない。そこを支えるために作っています。

AIはコードを書けます。難しいのは、複数のAIエージェントが並列に作業しても、プロダクトの意図、設計境界、検証証跡、レビュー証跡を失わないことです。VibePro はそれらを `.vibepro/` の成果物と機械的な Gate に変えます。

## 短い告知

VibePro をOSS公開に向けて整えています。

Codex や Claude Code に実装を任せても、プロダクト意図・品質基準・レビュー証跡が散らばらないようにするためのCLIです。

やることはシンプルです。

- 作りたいものを記録する
- AIに渡せる粒度へ分ける
- 必要な検証とレビューを要求する
- 証跡が足りなければPR作成を止める
- waiverや判断を会話ではなく成果物に残す

## 長文記事の構成

### 1. 問題

AIコーディングツールは、小さいタスクでは速い。

でもプロダクト開発はコード生成だけではありません。画面、API、データの流れ、性能、セキュリティ境界、状態遷移をAIが触り始めると、問題は「コードを書けるか」ではなく「意図したプロダクトとして成立しているか」に変わります。

多くの場合、人間はAIを見張ることで対処しています。

- 意図を何度も説明する
- 変更ファイルを手で追う
- 後からテストを頼む
- レビュー文脈をチャットに貼る
- PRを出していいか勘で判断する

これはAIエージェントを並列に使うほど破綻します。

### 2. VibePro がやること

VibePro は、作業の契約をリポジトリ内に残します。

- Story
- 設計
- 仕様
- タスクの引き渡し
- リスクに応じた Gate
- 検証証跡
- 役割別エージェントレビュー
- PR証跡
- 意思決定記録

必要な証跡が足りなければ、VibePro は作業を止めます。必要に応じて Codex / Claude Code のサブエージェントレビューを要求し、その結果を現在の git 状態に紐づけます。例外判断やwaiverも会話ではなく成果物に残します。

### 3. なぜ重要か

目的はAIにもっとコードを書かせることではありません。

AIにより多くの実装を任せながら、チームが次のものを握り続けることです。

- プロダクト意図
- 品質基準
- 責務境界
- 証跡
- マージ可能性

### 4. 試し方

npm公開前:

```bash
npm install -g git+https://github.com/Unson-LLC/vibepro.git
vibepro --help
```

npm公開後:

```bash
npx vibepro --help
```

Story があるリポジトリで:

```bash
vibepro pr prepare . --story-id <story-id> --base main
```

見るもの:

- `.vibepro/pr/<story-id>/review-cockpit.html`
- `.vibepro/pr/<story-id>/gate-dag.html`
- `.vibepro/pr/<story-id>/pr-body.md`
